2006 Aug 12th 看Youtube沒聲音

傷當神奇， 因為前幾天安裝某軟體時突然重開機， 這幾天電腦就怪怪的， 昨天是Audacity打不開， 今天是看Youtube時會沒有聲音， 但是foobar、Media Player Classic都很正常， 網路上搜尋之後，找到個解決方法， 真是神奇。 開啟Registry Editor， 到HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Drivers32， New a String Value named wavemapper with value msacm32.drv。 確定按下去後，聲音就跑出來了， Audacity也能開啟了

2006 Aug 10th iSCSI測試

Internet SCSI是由IETF提出的協定， iSCSI的好處是它架構在TCP/IP上， 利用現有普遍的Ethernet網卡就可以了， 不需要另外添購設備。

現在Windows XP上使用iSCSI相當簡單， 最近某種神秘需求已經確實可以利用iSCSI達到目的， 首先講點名詞， 一台電腦當做server，其他台電腦稱為client， 在client要安裝initiator， 目前微軟就有提供Initiator軟體， 請搜尋「Microsoft iSCSI Software Initiator」， 測試時安裝的版本是2.02，client只需要這個就夠了。 server端則需要軟體成為target， 這次測試使用的是Rocket Division Software 「StarWind iSCSI Target」， 測試版本為v3.1.4， 掛載映像檔則是使用Alcohol 120%， Daemon Tools尚未測試。

  • Alcohol 120%
  • StarWind iSCSI Target
  • Microsoft iSCSI Software Initiator

server端安裝好Alcohol 120%和StarWind iSCSI Target之後， 掛載需要的映像檔，在Alcohol 120%右下角畫面的光碟機按右鍵， 選「共用」，英文寫「Sharing」， 分享掛載映像檔那台，按「共用此裝置」，英文「New Share」， 看到那台寫共用即可，英文寫on。 StarWind基本上只要安裝起來即可， 可以不用動任何設定， 當然要改密碼也可以， 但是StarWind改過之後，Alcohol這邊也要改。

當server端正常地設定共用之後， client端請開啟Microsoft iSCSI Initiator， 在Discovery標籤的Target Portals，按Add， IP填server那台的IP，StarWind沒動過的話， port就不用動，預設為3260。 確定後，換到Target標籤， 此時應該server設定的device會出現在下方， Status為Inactive， 按下Log On，有改過密碼的話，要到Advanced去改， 不然就直接按下OK，Status顯示變成Connected就代表ok了， 到我的電腦去看，有沒有多一個光碟機， 有的話就大功告成，收工。

2006 Aug 9th Windows Live Spaces擷取相片

之前寫過MSN Spaces擷取程式， 這回改版變成Windows Live Spaces之後， 程式一跑，竟然不work， 趕快Ethereal開起來， 事後發現這個舉動還蠻不必要的， 因為沒有改很多東西， 不需要動用Ethereal。 這次更改的地方， 首先是tk.files.storage變成tkfiles.storage， 少了一個.(dot)， 再來下一頁的部分， 不需要用Regular Expression去抓spa_strNextUrl， 只要改網址的_c11_PhotoAlbum_photo參數， 從0開始，假設有178張，就是0到177， 其他就一樣。剛才測試一下，還蠻正常的。

又，題外話， Opera 9.01適應中…Ctrl+N變成另外獨立視窗， 開tab要用Ctrl+T，目前還是習慣按8.5的Ctrl+N， 改用Ctrl+T是為了和Firefox統一？

2006 Aug 9th 真糟糕…

整週每天都超過12點才睡，還有兩天超過3點才睡！ 而且原因是革新！是的，革新又出現了，這次是玩中文版， 如夢似幻劇本的伊達家，統一東北地方後，開始和德川家對決， 還得小心北条和上杉亂入(還好上杉是景勝，謙信公的話就不是單純亂入了~~)， 在米澤城戰了好幾回，現在是德川、羽柴都滅了，剩下四國和九州。

嗯…所以這幾天真是糜爛的生活…。 唯一比較有意義的是寫了「歌詞ナビ Retriever」， 不過這也是大概半小時就可以結束的東西 (其實半小時還算多？看Ethereal和版面設計花比較多時間…)

測了一下，發現有趣的事， 歌詞ナビ必須使用post來抓， 而utamap只需要get，也就是在網址上打就可以， 但是歌詞ナビ抓下來的會斷行， utamap的就不會。傷當神奇， 兩個都是LF，為什麼會有這樣的結果呢， Ethereal開起來看， 原來utamap傳回來的是text/html， 而歌詞ナビ的是text/plain。 後製處理是一個方法，還可以把開頭的參數名稱拿掉， 但是純粹瀏覽器來做的方法似乎不太可行， 要拿遠端的內容會是security violatoin啊

補充：XMLHttpRequest can't access remote servers. XMLHttpRequest cannot have a remote server as a target. This is a basic cross browser security rule that may not be immediately obvious to the newcomer to Ajax development.
