2006 Jul 31st 雜

又好幾天沒更新了…話說這幾天沒什麼成就感。 MiKTeX使用utf8編碼， 目前是弄到\begin{CJK}{UTF8}{}可以產生pdf檔出來了， 但是手續還蠻麻煩的， 除了網路上可以找到的cyberbit字型匯入之外， 還要抓CJKutf8.sty，而程式要用兩段式， 也就是先latex再dvipdfmx， 沒辦法直接用pdflatex， 目前產生出來的pdf檔， 中文的部分不能copy and paste， 看網路上在Linux下的好像都可以啊…

另外是研究網路電視， 所謂的P2P Streaming， 現在目標放在「PPLive」和「沸点网絡电视」這兩套軟體。 蠻有名氣的「PPStream」，我到現在好像只成功看過一次吧， 而且掛AppLocale也不一定能完全正常顯示， 頻道列表又一直自動更新(亂跳…)， 所以目前不考慮。 PPLive現在有paper在研究， 而且程式界面寫的不錯， 不需要用AppLocale就可以正常顯示， 還可以選擇繁中、簡中、English。 沸点，寫到一半發現出新版了， 嗯…前幾天測試舊版時， 軟體本身只有提供所謂的TV engine， 沒有內建Player功能， 實際使用時會把Windows Media Player叫出來， 也可以設定用別的Player， 界面必須使用AppLocale來看簡中， 頻道數比PPLive少，播放順暢度還不錯， 而新版剛剛試一下，做了內建Player， 頻道好像有多一點，還是要用AppLocale。 另外還有一個TVUPlayer在測試名單中， 這個的好處是有不同上述兩者的頻道， 有比較多英文台，例如ESPN, HBO， 似乎和另一個軟體SopCast有關係？(之前Ethereal有看到sopcast這個字出現)。 努力整理出頭緒來…沸点的windump log還沒處理。

昨天下午、晚上都是在寫擷取程式， Portable Playlist 這個網站可以把mp3的連結放在上面， 而實際運作是用Ajax去資料庫讀網址再餵給用Flash寫的Player來播放， 網址新增後沒有辦法修改， 所以理論上從瀏覽器是沒辦法看到網址， 但是Ethereal開起來就一清兩楚， 擷取程式是寫自動把全部的網址都抓下來， 雖然到現在還是搞不懂ASP.NET_SessionId是如何來的， 要擷取全部的網址是寫好了， session id就得手動去Ethereal抓。

待辦事項： xslt and xml for `that' website , learning about Linux Kernel , IPTV paper study。 寫是這樣寫啦，真正完成會有多少呢…

2006 Jul 18th 真．三國無雙4 Special PC

終於全部的道具都拿到了，來寫一下心得。 這代剛開始玩的第一個感覺就是，比真三三玩起來還順， 硬體條件沒有變更的情形下， 這是為什麼呢？不久後就發現啦， 有修過Computer Graphics就會知道， 在畫小兵的時候，要等武將靠近到一定距離後再畫出來， 而這次PC上的四代，把特效全開後， 畫小兵的距離比三代近很多， 所以運算量減少很多，畫面自然就比較順。

武將方面，這次新增角色分別有： 星彩、關平、曹丕、龐德、凌統、左慈， 其中左慈是大魔王，如果是玩PS2版的話， 要先讓其他武將都出現後，才能使用左慈， 而PC版是PS2的真三四加猛將傳， 一開始就能使用全部武將，也能打全部關卡； 既然有大魔王，當然就先練這隻了， 其實最最一開始，是想練趙雲， 當初真三三PC版也是先練趙雲， 但是打了一場之後，不知道是因為操作尚未適應， 還是四代趙雲改弱不少，用起來傷當不順手， 想說試一下左慈，喔！一試就上手， unique武器也不會太難拿，雖然是要打敗呂布， 不過虎牢關的等級比較低，還算簡單。 相較之下，凌統的unique武器就比較辛苦， 要打敗張遼四次，而且都是陽炎狀態…。 凌統在這代裡也算作弊角色，本身攻速非常快， C3單挑武將用，可轉向； C4範圍技，強到可以無腦使用(就是狂放C4，不需要想太多)。 左慈和凌統就是先練滿的兩隻， 但是說到真三國無雙，傳統強將當然非小強莫屬， 三代的時候沒有用過， 這次一玩才知道…真的是無腦C3， 可以說靠一招C3就打遍全場啊， 偶而想到再放一下無雙就好了。

除了上述三隻，無雙模式有全破的是： 甄宓、星彩、姜維、馬超， 其中星彩是easy過的，其實沒什麼練， 另外三個是hard。 甄宓就是「打小兵集氣->放無雙->打小兵集氣」循環； 姜維的C3還不錯，要小心背後就是了； 馬超是用馬上超人打法，就是騎絕影按charge。 有玩幾次，無雙模式還沒全破的有： 月英、關平， 月英的C3連兩次就能接到30段，不過目前尚未用順手； 關平的unique武器是輕重量， C3最後一下震波有破防，所以不用裝陽玉。

2006 Jul 15th HTML comment的規定

原來HTML comment也是有學問的， 一般我們熟知的HTML comment就是『<!--』開頭、『-->』結尾， 然後在comment裡不可以出現「-->」， 因為會被判定為comment結束。 然而如果在comment中出現「-」的話呢？ w3c的說法：結尾的「--」和「>」中間可以有空白， 所以在comment中出現兩次或更多的「-」， 會造成問題，應該要避免； Web Design Group的說法： 合法的SGML comment， 『<!』代表區塊開始， 『>』代表區塊結束， 在區塊中有零至多個comment， comment的開始與結束都是用『--』來表示， 所以「<!------> hello-->」是合法的comment， 共有兩個comment，分別是「----」和「-->hello--」， 但是這樣寫的話，在瀏覽器裡會看到「hello-->」， 事實上editor的highlight就已經告訴你了

結論，開頭『<!--』、結尾『-->』， 中間不要出現「>」和「--」(「---」也算「--」的variant喔)

2006 Jul 13th 用cl和link製作dll

關係到前(錢)途的重要功能， 總於寫出來了！(？) 

// dll.c
#include <stdio.h>

void hello(int n, char *s){
    fwrite(s, 1, n, stdout);
}
dll.def 
LIBRARY dll

EXPORTS
hello @1

rem  compile.bat
cl.exe /W3 /c dll.c
link.exe /OUT:"dll.dll" /DLL /def:".\dll.def" dll.obj

然後會產生出dll.dll檔，用Python加ctypes(要另外安裝)， Python code as the following: 

import ctypes
dyn=ctypes.cdll.LoadLibrary("./dll.dll")
dyn.hello(6, 'hello\n')

基本上，離實用距離還很遠， 但是咕了好幾天，segmentation fault不知道幾次， 快把VS折片的情形下，終於有可以跑的東西出現…。 參考網址：

2006 Jul 2nd 下載與轉檔flv

轉檔使用Riva FLV Encoder， 使用方法： 選擇Input Video， 把Output Video的副檔名換成avi或是mpg， 按下Encode即可， 目前轉youtube的檔，Resolution選320x240似乎是比較好的選擇。 需要注意的是，轉檔後的品質通常會再下降， 原本品質就不是很好的flv檔，轉完之後，有可能會糊到一種難以接受的境界…

下載部分，目前有不少網站可以把下載位置抓出來， 例如這裡。 當然也可以自己寫， youtube的話，從網頁擷取出兩個參數：video_id和t， 前面接上http://youtube.com/get_video就可以了， 在該網頁搜尋watch_fullscreen或是player2.swf就會看到那兩個參數
